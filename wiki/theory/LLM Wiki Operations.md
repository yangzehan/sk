# LLM Wiki Operations 运营

## 三个核心操作

### 1. 吞下 (Ingest)

将新源放入 raw/，让 LLM 处理：

**示例流程：**
1. LLM 阅读源代码
2. 与你讨论关键要点
3. 在 wiki 中写摘要页
4. 更新索引
5. 更新相关实体和概念页面
6. 在日志中添加条目

**一个来源可能覆盖 10-15 个 Wiki 页面**

建议：
- 一次只吸收一个资料，保持参与
- 阅读摘要，查看更新，指导 LLM 强调什么
- 也可以批量吸收多个来源

### 2. 查询 (Query)

对着 Wiki 提问：
- LLM 搜索相关页面
- 阅读并综合答案，带引用
- 答案可以是：标记页、比较表、幻灯片、图表

**重要：好的答案可以作为新页面归档到 Wiki**

- 你要求的比较、分析、联系
- 很有价值，不应消失在聊天记录
- 让探索像资源一样积累知识库

### 3. 绒毛 (Groom/巡检)

定期让 LLM 对 Wiki 进行健康检查：

**检查项目：**
- 页面间矛盾
- 陈旧说法（被新来源取代）
- 孤立页面（无入站链接）
- 提及重要概念但缺乏独立页面
- 缺失交叉引用
- 可用网络搜索填补的数据空白

LLM 擅长提出新问题和寻找新资源。

## CLI 工具（可选）

- **QMD**: 本地搜索引擎，BM25/矢量搜索 + LLM 重排序
- 自定义简单搜索脚本

## 技巧与窍门

- **Obsidian Web Clipper**: 网页转 markdown
- **本地下载图片**: 设置附件文件夹路径
- **Obsidian 图表视图**: 了解 Wiki 结构
- **Marp**: Markdown 幻灯片
- **Dataview**: 动态查询
- **Git**: 版本历史、分支、协作
